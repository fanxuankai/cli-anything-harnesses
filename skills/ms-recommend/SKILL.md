# ms Recommend

## 概述

通过本地 `cli-anything-ms` CLI 处理影视推荐浏览请求。
处理时先弄清楚用户想看哪个推荐来源、电影还是电视剧，再读取这个来源当前支持的筛选项，最后再取推荐条目。不要直接调用后端接口，也不要顺手订阅。

## 安装

```bash
pip install cli-anything-ms
```

## 触发场景

- `影视推荐`
- `推荐电影`
- `豆瓣电影推荐`
- `TMDB 高分电影推荐`
- `推荐 2025 年动作片`
- `推荐豆瓣电视剧，近期热度，华语`

## 工作流

### 1. 先确认推荐来源

如果用户没有明确说明平台，先执行：

```bash
cli-anything-ms --json media recommend sources
```

把可选来源用中文告诉用户，请用户选一个。

如果用户已经说了平台，就按 CLI 支持的来源名调用。例如：

- `豆瓣` -> `douban`
- `TMDB` -> `tmdb`

### 2. 再确认电影还是电视剧

来源确定后执行：

```bash
cli-anything-ms --json media recommend channels --source <alias>
```

如果用户没说清楚要电影还是电视剧，就把可选频道列出来让用户确认，不要猜。

### 3. 根据当前来源读取筛选项

频道确定后执行：

```bash
cli-anything-ms --json media recommend options --source <alias> --channel <channel>
```

必须先读当前来源和频道实际支持的筛选项，不要凭记忆硬编码。
用户提到“高分优先”“2025”“动作”这类偏好时，先在返回的筛选项里找对应文字，再把它们带进推荐请求。用户没提到的筛选项保持为空，但仍然要放进请求里，避免漏掉 CLI 需要的选项。

如果当前来源不支持用户想要的条件，就直接说明这个条件不可用，请用户换一个筛选方式。

### 4. 获取推荐条目

筛选条件准备好后执行：

```bash
cli-anything-ms --json media recommend items --source <alias> --channel <channel> --options '<json>' --page <n> --page-size <n>
```

分页按用户说法处理：

- 默认 `page=1`
- 默认 `page-size=10`
- 用户说“前 N 个”，就取 N 个
- 用户要求看第 N 页，就取对应页

### 5. 用自然语言展示结果

先说明这次推荐大致按什么条件筛出来。每个条目用正常中文介绍，包含片名、年份、类型、评分、是否已订阅、是否已入库和海报链接即可。不要写成字段说明，也不要把内部字段名放进正文。

如果用户明确要求看原始结果，再补充原始 JSON。

## 注意事项

- 这个技能只做推荐浏览，不继续执行订阅。
- 不跳过筛选项读取步骤直接猜平台参数。
- 不直接调用后端接口。
