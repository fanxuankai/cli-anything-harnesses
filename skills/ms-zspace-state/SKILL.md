# MS Zspace State

## 概述

用本地 `cli-anything-ms` 获取极空间当前系统状态，并把原始结构化数据整理成简洁的中文结论。

## 工作流

### 步骤 1：确保 `cli-anything-ms` 可用

- 先检查本机是否可以直接执行 `cli-anything-ms`。
- 如果命令不可用，再执行：

```bash
pip install cli-anything-ms
```

- 如果安装失败，直接转述安装失败原因，停止后续步骤。

### 步骤 2：调用极空间状态接口

- 固定执行下面这条命令，不要改成别的接口：

```bash
cli-anything-ms --json plugin call --code zspace_service_assistant --body '{"action":"get_recent_state","body":{}}'
```

- 不要要求用户手动拼接参数。
- 不要调用别的 Media Saber 接口。

### 步骤 3：解析返回结构

- 顶层是 Media Saber 的标准响应外壳。
- 真实的极空间状态在插件层返回对象的 `data` 列表中。
- 默认取列表中的最新一条状态记录作为当前状态。
- 如果命令返回空列表，明确告诉用户当前没有拿到极空间状态数据。
- 如果返回结构不符合预期，明确说明“已调用成功，但状态数据结构异常”。

### 步骤 4：提取重点信息

- 优先提取这些字段并总结：
  - 设备名称、型号、运行时长
  - CPU 名称、占用、温度
  - 内存总量、使用率
  - GPU 名称、占用、温度（如果有）
  - 网络上下行速度
  - 风扇转速
  - HDD 和 SSD 的容量、已用空间、温度、状态

### 步骤 5：总结并回复

- 先给一句总体状态概述，例如“当前整体运行正常”。
- 再补充关键指标摘要。
- 如果用户只问某一类指标，例如 CPU、内存、磁盘温度，就优先回答该部分，同时保留一句整体判断。

## 异常提示规则

- 如果 CPU、GPU、磁盘或 SSD 温度明显偏高，要显式提示关注。
- 如果内存使用率较高，要显式提示。
- 如果某块磁盘或 SSD 的 `state` 不是正常状态，要点名指出。
- 如果插件调用失败，例如“还没有安装秘籍”“插件未启用”“请求超时”，直接转述后端失败原因，不要编造系统状态。

## 注意事项

- 不增加历史对比。
- 不做趋势分析。
- 不主动给出修复建议清单，除非用户继续追问。
- 输出以中文总结为主。
