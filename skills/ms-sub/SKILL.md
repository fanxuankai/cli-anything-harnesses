# ms Sub

## 概述

通过本地 `cli-anything-ms` CLI 处理电影和电视剧的搜索、订阅列表查看与新增订阅。
这个技能负责三件事：

- 搜索 TMDB 候选
- 分页查看现有订阅
- 在用户明确要求时新增订阅

## 安装

```bash
pip install cli-anything-ms
```

## 功能

### 搜索影视

- 适用于：
    - `搜索电影流浪地球`
    - `搜一下电视剧西游记`
    - `先搜一下`
    - `先看下结果`
- 每次搜索前都要先确认用户要找电影还是电视剧：
    - `电影` -> `movie`
    - `电视剧` -> `tv`
- 如果没说清楚，不要猜，先让用户补充。
- 去掉外壳词和可选的季数描述，只保留真正用于 TMDB 搜索的标题部分。
- 只搜索 TMDB：

  ```bash
  cli-anything-ms --json media search --source tmdb --keyword "<title>" --page-size 5
  ```

- 先按电影或电视剧过滤结果。
- 选择候选时，优先找标题最接近用户输入的结果。只有一个明显匹配时可以直接使用；多个都合理时，展示最多 3 个候选让用户确认；没有匹配时，直接说明没找到。
- 只搜索时，在展示候选后停止，不要自动进入订阅流程。
- 搜索结果用自然中文展示，说明标题、年份、类型、评分、是否已订阅、是否已入库和海报链接即可。不要把内部字段名写给用户看。

### 订阅影视

- 适用于：
    - `订阅电影流浪地球`
    - `订阅电视剧西游记`
    - `检查 XXX 有没有订阅，没有就订阅`
- 订阅流程必须建立在 TMDB 搜索结果之上，不要跳过搜索直接订阅。
- 订阅前同时看候选是否已经订阅、是否已经入库。已经订阅就直接提示；已经入库但没订阅时，默认不要再新增订阅；只有既没订阅也没入库时，才继续新增订阅。
- 如果已经订阅，回复简短确认，例如 `ℹ️ 已订阅`。
- 如果未订阅但已经入库，默认不要继续订阅，明确告诉用户例如 `ℹ️ 已入库，默认不再新增订阅`。
- 只有用户明确坚持“已入库也要订阅”时，才允许继续执行新增订阅。
- 如果尚未订阅，执行：
    - 电影

      ```bash
      cli-anything-ms --json subscribe add --type movie --name "<title>" --year <year>
      ```

    - 电视剧

      ```bash
      cli-anything-ms --json subscribe add --type tv --name "<title>" --year <year> --season <season>
      ```

- 如果用户没有明确指定电视剧季数，默认使用第 `1` 季。
- 调用新增订阅时，优先使用 TMDB 搜索结果里的规范片名和年份。
- 成功新增后，统一回复 `✅ 订阅成功`。

### 查看订阅

- 适用于：
    - `看看电影订阅`
    - `查看电视剧订阅`
    - `电影订阅列表`
    - `电视剧订阅前 20 个`
- 直接执行：

  ```bash
  cli-anything-ms --json subscribe page --type <movie|tv> --page <n> --page-size <n>
  ```

- 分页按用户说法处理：
    - 默认 `page=1`
    - 默认 `page-size=20`
    - 用户说“前 N 个”，就取 N 个
    - 用户说“第 N 页”，就取对应页
- 输出订阅列表时，用自然中文说明名称、年份、类型、季数、状态和 TMDB 信息。状态要翻译成中文，不要只给数字。
- 如果用户明确要求原始结果，再补充原始 JSON。
- 查看订阅时，不要先做 TMDB 搜索。

## 注意事项

- TMDB 是唯一搜索源，不要搜索豆瓣。
- 用户是查看订阅列表时，优先走 `subscribe page`，不要先做 TMDB 搜索。
- 用户只是想搜索时，不要自动进入订阅流程。
- 不要暴露或提及 `subscribe config`；`subscribe add` 内部已经会处理默认配置。
- 如果一句话里有多个搜索、查看订阅或订阅意图，要逐个处理，并逐条返回结果。
- 订阅前不要只看是否已订阅，还要一起考虑是否已入库。
- 新增订阅时，优先使用 TMDB 结果里的规范片名和年份。
- 如果新增失败，例如后端识别失败或业务校验失败，要清楚转述失败原因，不要误报成功。
